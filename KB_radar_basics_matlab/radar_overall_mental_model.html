雷达信号处理
整体心智模型

把已经学到的东西收拢成一张"地图"：从现实世界的雷达体制，到把电磁波抽象成信号， 再到信号处理的目标与方法，最后标出还没踏足的疆域（暂不含 SAR）。 核心线索始终是那个三维数据立方体 x[fast time, slow time, array element]。

01

从这张图能"感受"到的五件事

心智锚点

  • 主动感知：波形是你自己发的，所以你知道发的是什么——这正是匹配滤波 / 脉冲压缩能成立的根本前提（被动感知没有这个先验）。
  • 抽象层是一摞：硬件框图（这张图）→ 雷达方程（功率/距离预算）→ 信号模型 x[·,·,·] → 处理算法。你目前学的全部落在最后两层，且物理上都发生在那个高亮的 Digital Signal-and-Data Processor 盒子里。
  • 收发对称：每一级（数字/模拟/射频）都成对存在 TX↑ 与 RX↓，相控阵是唯一与物理空间相接的接口。

"Radar Scene" 是你要反演的东西

  • 环境里你真正关心的是一组散射体，每个由 \((R, v, \theta, \phi, \alpha)\) 描述：距离、径向速度、方位、俯仰、复反射率。
  • 收到的从来不是"干净目标"，而是叠加：目标回波 + 杂波（地/海/雨）+ 干扰机 + 热噪声。三条信道（forward/backward/interfering）正对应这三类来源。
  • 底部那排 Scheduler / Optimizer / AI 在提示一个更高的层次——认知雷达：雷达不仅处理回波，还能反过来自适应地决定下一刻发什么波形、扫哪个波束。（这部分你还没学，见 §4）
02

数学抽象：把传播 / 散射 / 反射压缩成一个信号模型

第二层抽象做的事情是：电磁波的传播、在目标上的散射与反射，这些连续的物理过程， 最终在接收机里只留下三种相位结构。一个点目标对回波做的全部"标记"，就是给基带波形乘上三个复指数：

$$ x[\,\underbrace{l}_{\text{fast}},\ \underbrace{n}_{\text{slow}},\ \underbrace{m}_{\text{elem}}\,] = \alpha \cdot \underbrace{u\!\left(\tfrac{l}{f_s}-\tfrac{2R}{c}\right)}_{\text{range delay}} \cdot \underbrace{e^{\,j2\pi f_D\, n T_r}}_{\text{Doppler phase}} \cdot \underbrace{e^{-j2\pi m\, d\sin\theta/\lambda}}_{\text{spatial phase}} \;+\; w[l,n,m] $$
range delay · 距离 → 延迟（fast time） Doppler phase · 速度 → 慢时间相位（slow time） spatial phase · 角度 → 空间相位（array element）

其中 \(\tau=\dfrac{2R}{c}\)（往返延迟），\(f_D=\dfrac{2v}{\lambda}\)（多普勒频率），\(T_r=1/\mathrm{PRF}\)（脉冲重复周期）， \(u(\cdot)\) 是发射脉冲包络（LFM），\(\alpha\) 是复幅度，\(w\) 是噪声。多目标就是对 \(q\) 求和：\(\sum_q \alpha_q(\cdots)\)。

为什么这一个式子"全面且准确"

  • 可分离（separable）：三个相位各自只挂在一个维度上 —— 这就是能把三维处理拆成三次一维处理的根本原因。
  • 每个点目标 = 一个 rank-1 时空结构 \(\;\alpha\,\mathbf a(\theta)\,\mathbf b(f_D)^{\!\top}\)：\(\mathbf a\) 是空间导向矢量，\(\mathbf b\) 是慢时间（多普勒）导向矢量。处理 = 用对应模板去投影。
  • 可扩展：把单个 \(\sin\theta\) 换成 URA 的两个方向余弦 \((\xi,\eta)\)，就从一维 ULA 升级成二维测角（方位+俯仰），结构不变。

一个必须时刻警惕的点：符号约定

  • 本课程 ULA 约定：\(u=\dfrac{d\sin\theta}{\lambda}\)，逐阵元相位 \(\Delta\phi=-2\pi u\)；做空间 FFT 时对 conj(arraySnapshot) 取 FFT，峰才落在正 \(u\)。
  • 匹配滤波、Doppler FFT、波束形成模板里的共轭 / 正负号极易写反 —— 这是你当前仍需巩固的薄弱点。

三个"极端实验"值得亲手拖一下：①把 vr 拖到 ±150 m/s，相量点会塌缩重叠 —— 这就是多普勒混叠（超过无模糊速度 ±75 m/s）； ②把 θ 拖大、并切到 d = λ，当 \(|u|>0.5\) 越过空间 Nyquist，会触发栅瓣告警； ③单独改 R，只有距离门和脉内 blip 在动，速度/角度卡片纹丝不动 —— 这就是"可分离"的直观证据。

升级到三维空间：2D 相控阵（方位 θ + 俯仰 φ）

把 ULA 沿第二个方向也铺开，就得到一个 8×8 平面阵（URA），放在 \(x\text{–}y\) 平面、boresight 指向 \(+z\)。 此时目标方向需要两个角：方位 θ 与俯仰 φ（本课程约定 φ=90° 正对 boresight，φ=0° 为掠射 horizon）。 阵列测的是两个空间频率 \(\omega_x,\omega_y\)，对应两个方向余弦 \(\xi=\cos\phi\cos\theta\)、\(\eta=\cos\phi\sin\theta\)。 下面是一张三维示意（等距投影）：左侧拖动 θ/φ，看阵面上的相位锥（phase taper）怎样倾斜，右侧是同一阵面的相位热力图。

两个直觉实验：①把 φ 拖到 90°（正对 boresight），ξ=η=0，整片阵面变成同一个颜色 —— 所有阵元同相、无需相位补偿； ②固定 φ<90° 只转 θ，相位锥的倾斜方向随之旋转 —— 这正是二维电扫在做的事：用 \((\xi,\eta)\) 两个空间频率把波束指向 \((\theta,\phi)\)。 ULA 是这张图在 \(\eta=0\)（单行）时的退化特例。

03

处理的目标与方法：从数据立方体到目标清单

High level：信号处理到底想干什么

一句话概括：把混着噪声、杂波、干扰的数据立方体，变成一张带"可量化误差"的目标清单（最终是航迹）。 它分解成四个相互衔接的子目标：

① 攒能量 (integration)

把分散在 fast/slow/array 三维里的目标能量相干叠加，最大化 SNR / SCNR。手段：匹配滤波、Doppler 积累、波束形成。这是后面一切的前提。

② 判有无 (detection)

在可控的虚警率 \(P_{fa}\) 下逐 cell 判决"有/无目标"。手段：CFAR。核心价值是给下游一个可标定的错误统计，这是人眼盯屏幕给不了的。

③ 估参数 (estimation)

对检出的目标估计 \((R,v,\theta)\)：峰值定位 + bin 内插。更高精度/分辨率用 MUSIC、MVDR/Capon。

④ 抑干扰 (suppression)

把杂波/干扰从目标里分离：慢时间上 MTI 对消零多普勒杂波；阵元上 MVDR 自适应置零干扰。(动平台的空时联合抑制 STAP 还没学，见 §4)

可视化：数据立方体 → 目标清单（信息如何逐级被压缩）

整条链路可以看成一台信息提炼机：左端是每个 CPI 数 MB–GB 的原始三维立方体， 每经一级处理就降维 + 压缩一次，最终右端只剩几条带误差标定的目标记录，再连成航迹。

读法：距离 (fast)、多普勒 (slow)、 角度 (elem) 三个维度在前几级被逐一"消化"成参数轴； 检测把连续统计量切成 0/1；后处理再把 mask 提炼成带误差标定的清单。 关键差别：人眼也能在 RD 图上看出亮点，但只有算法能给出可量化、可控的 \(P_{fa}\)/\(P_d\) 交给跟踪器 —— 这正是 §2 那个信号模型存在的意义。

04

还没学到的方面（不含 SAR）

你已经走通了"单/多点目标 → RD → 阵列测角 → 超分辨/自适应 → CFAR → MTI"这条主线。 下面是同一张地图上仍空白的区域，按主题分组，并标出它在前面框图 / 处理链里的位置。

A · 波形设计与模糊函数

  • 模糊函数 \(\chi(\tau,f_D)\) —— 把距离分辨率、速度分辨率、旁瓣统一在一张图里的"母工具"，你目前是分开学的。
  • 相位编码波形（Barker、P 码、多相码）、步进频；脉压旁瓣抑制（加权 / 失配滤波）。
  • 距离–多普勒耦合（LFM 的固有斜置）；多 PRF / staggered PRF 解距离-速度模糊。

B · 检测理论的纵深

  • Neyman–Pearson 准则 / 似然比检验：从原理推出"匹配滤波就是最优检测器"。
  • 相干 vs 非相干积累及其积累增益；二进制 / M-of-N 积累。
  • Swerling 起伏目标模型（0/1/2/3/4）—— RCS 随机起伏如何改变 P_d。
  • GO / SO / OS-CFAR 的具体实现（你已知其名与动机，还没动手）。

C · 估计理论与精度极限

  • Cramér–Rao 下界（CRLB）：测距/测速/测角能达到的理论精度极限。
  • 最大似然估计；单脉冲测角（monopulse，和/差波束）——亚波束宽度的高精度测角。
  • 其它超分辨：ESPRIT、压缩感知 / 稀疏重构；bin 内插做亚分辨估计。

D · 杂波与空时处理（重点缺口）

  • 杂波统计模型（Rayleigh / Weibull / K 分布 / 对数正态）—— 真实杂波非高斯。
  • MTI 盲速与 staggered PRF；MTD（多普勒滤波器组）。
  • STAP 空时自适应处理 —— 动平台（机载/星载）联合 角度–多普勒 抑杂波，是这条线最大的一块。对应"clutter ridge"的认识。

E · 阵列与波束的纵深

  • 二维平面阵 / URA（方位 + 俯仰），你目前只做了 1D ULA。
  • 子阵、数字波束形成（DBF）架构；LCMV / GSC 等自适应变体。
  • 阵列校准、互耦；波束调度与扫描策略（对应图中 Resource Scheduler）。

F · MIMO 雷达

  • 正交 / 波形分集发射，接收端合成虚拟阵列，用更少阵元换更大等效孔径。
  • 与相控阵的取舍：自由度 vs 相干发射增益。

G · 系统级与硬件非理想

  • 雷达方程 / 链路预算：SNR ∝ 1/R⁴，噪声系数、损耗、灵敏度 —— "我能看多远"的那一层。
  • RF / ADC 非理想性：相位噪声、I/Q 失衡、量化；eclipsing、straddling loss。每个都回答同一个问题：它如何污染 RDA cube。

H · 数据处理与认知雷达

  • 跟踪：α-β / Kalman / IMM 滤波；数据关联（NN / PDA / JPDA / MHT）；航迹起始与维持。这是框图里 "Data Processor" 的后半段。
  • 认知雷达：资源调度 + 凸优化波形/波束设计 + AI —— 对应图底部那排子系统，也接得上你自己提的"环境学习杂波检测"研究方向。

若要排个优先级建议：模糊函数（A）会把你已经分开掌握的三种分辨率收口成一个统一工具； STAP（D）是阵列 + 多普勒两条线的自然汇合点、含金量最高； 跟踪（H）则让整条链路第一次真正闭环到"航迹"。
