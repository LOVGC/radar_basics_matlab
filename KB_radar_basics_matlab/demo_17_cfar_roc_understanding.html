如何评价一个 detector：不是一个数字，而是一条 tradeoff 曲线

Demo 17 的核心不是“哪个 CFAR detector 分数最高”，而是把检测器放在具体工作条件下看： 固定 SNR 和设计 Pfa，用 Monte Carlo 估计 Pd。 扫描不同条件后，就得到 (SNR, design Pfa) -> empirical Pd 这张检测性能地图。

01

你的 Demo 17 理解

Experiment Point

每一个点固定两个条件

先固定一个目标 SNR，再固定一个 CFAR 的设计参数 design Pfa。这个 Pfa 决定 threshold。

given (SNR, design Pfa) -> run many trials
Target Present

用 Monte Carlo 估计 Pd

每次 trial 都生成“有目标”的随机观测，然后看目标附近是否被 CFAR 检出。 检出的比例就是 empirical Pd。

Pd = detected target trials / target-present trials
Sweep

扫描 SNR 和 Pfa

改变 SNR 和 design Pfa，就得到很多 (SNR, Pfa, Pd) 点。把它们连起来，就是 ROC-style 曲线。

Pd = f(SNR | design Pfa)
02

ROC 思维是什么

Classic ROC

固定 SNR，扫 threshold

经典 ROC 通常固定目标强度和环境，然后移动 threshold，观察 Pfa 和 Pd 如何一起变化。 threshold 降低时，Pd 上升，但 Pfa 也会上升。

classic ROC: Pd = f(Pfa | fixed SNR)
Demo 17

固定 design Pfa，扫 SNR

Demo 17 采用另一个等价视角：先选一个 false-alarm operating point， 再问不同 SNR 下能得到多少 Pd。 所以它是一组 ROC-style 的 Pd vs SNR 曲线。

Demo 17: Pd = f(SNR | design Pfa)
Operating Point

真正的问题是选工作点

雷达 detector 设计不是追求一个孤立分数，而是在任务约束下选工作点： tracker 能承受多少 false alarms，目标 SNR 大概多低，任务要求多少 Pd。

choose Pfa, then check required SNR for target Pd

Demo 17 的数值结论

Design Pfa Threshold 严格程度 约达到 Pd ≥ 0.9 所需 SNR 解释
1e-2 最宽松 约 10 dB 最容易检出弱目标，但 false alarms 也最多。
1e-3 较宽松 约 12 dB 比 1e-2 更少误警，但弱目标需要更高 SNR。
1e-5 严格 约 14 dB 误警率低很多，检测曲线明显右移。
1e-7 最严格 约 16 dB 低误警要求很强，但弱目标最容易漏检。
05

你现在应该带走的 mental model

Evaluation

Detector 没有脱离条件的“好”

评价 detector 时，至少要说明环境、SNR、false-alarm requirement 和检测概率。 单独说“这个 detector 好”是不完整的。

Tradeoff

Pfa 和 Pd 是阈值两侧的代价

Pfa 要求越小，threshold 越高，误警越少；但在低 SNR 时， 真目标也更难过 threshold，所以 Pd 会下降。

SNR

SNR 不直接设置 Pfa

Pfa 是 detector 的设计目标；SNR 主要影响目标存在时能不能超过这个阈值。 高 SNR 让严格 Pfa 下仍保持高 Pd 变得更容易。

Step 1

选择 SNR 比如 0 dB、6 dB、12 dB。

Step 2

选择 design Pfa 比如 1e-3 或 1e-7。

Step 3

运行 target-present trials 重复随机实验，统计目标被检出的比例。

Step 4

得到 empirical Pd 这个点就是 (SNR, Pfa, Pd)。

Step 5

用 noise-only trials 验 Pfa 检查实际误警率是否接近设计值。
